🔐 Auth / Login
POST/api/auth/loginLogin — get JWT token
Request Body
{"username": "admin", "password": "Admin@1234"}
Response
{"success":true,"data":{"token":"eyJ...","userId":1,"name":"Admin","role":"Admin","menuAccess":{...},"expiresAt":"..."}}
GET/api/auth/profileGet logged-in user profile
Response
{"success":true,"data":{"id":1,"name":"Admin","username":"admin","role":"Admin","status":"Active","menuAccess":{...}}}
PUT/api/auth/profileUpdate own profile
Request Body
{"name":"Admin User","contact":"0501234567","email":"admin@tfms.ae"}
POST/api/auth/change-passwordChange own password
Request Body
{"currentPassword":"Admin@1234","newPassword":"New@1234","confirmPassword":"New@1234"}
POST/api/auth/refresh-tokenRefresh JWT token (no body needed)

No request body needed. Uses current token from Authorization header.

GET/api/auth/meQuick check — user info from JWT (no DB call)
{"success":true,"data":{"userId":1,"username":"admin","role":"Admin","isAdmin":"True"}}
POST/api/auth/logoutLogout

JWT stateless — client discards token. Server records logout timestamp.

📋 Quick Reference
Payment Status Values:

Paid — paidAmount ≥ amount   Partial — 0 < paidAmount < amount   Pending — paidAmount = 0   Overdue — past due date & unpaid

Contract Status: Active | Expired | Terminated
Expense Nature: HO (Head Office) | Camp
TxnRecord Type: DR (Debit/Income) | CR (Credit/Expense)
Tenant Type: Individual | Company
All list APIs: Omit PageSize or set 0 = return ALL records
